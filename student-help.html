How to Submit GitHub Projects

  1. Open the assignment link from the approved classroom system.
  2. Accept the GitHub Classroom assignment.
  3. Complete the required files in your repository.
  4. Update the README reflection.
  5. Add screenshots, photos, audio files, or code evidence when required.
  6. Commit your changes.

Common GitHub Project Folders

  • README.md: explanation, reflection, and project notes
  • code: code files or copied code evidence
  • images: screenshots, photos, or visual evidence
  • audio: music or sound files when required

Codespaces Reminder

For web and game projects, students may use GitHub Codespaces to edit and test code. Always commit changes before closing. Stop or close unused Codespaces when finished.

Getting Help

Ask for help early. Be ready to show your code, explain what you tried, and describe what is not working.
