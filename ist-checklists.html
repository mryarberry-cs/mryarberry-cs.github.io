How to Submit GitHub Projects in Google Classroom

  1. Open the GitHub Classroom assignment link from Google Classroom.
  2. Accept the GitHub Classroom assignment.
  3. Complete the required files in your GitHub repository.
  4. Complete the README reflection.
  5. Add required code, screenshots, photos, audio, or evidence files.
  6. Commit your final changes in GitHub.
  7. Copy the link to your GitHub repository.
  8. Return to the matching Google Classroom assignment.
  9. Submit your GitHub repository link in Google Classroom.
  10. Mark the assignment as done.

Reminder: Your project is not finished until your final changes are committed in GitHub and your repository link is submitted in Google Classroom.

Universal Submission Reminder

Before submitting any IST GitHub project, make sure:

  • Your README is completed.
  • Your required code files are saved in the correct folder.
  • Your screenshots, photos, audio, or evidence files are included when required.
  • Your file names are clear and easy to understand.
  • You committed your final changes.
  • You can explain what you made and how it works.

IST 01 GitHub Intro

  • README.md is completed.
  • Sentence starters are replaced with my own answers.
  • I committed my changes.
  • I can explain what a repository and commit are.

IST 02 Portfolio Setup

  • README.md is completed.
  • About Me section is filled in.
  • Goals section is filled in.
  • Portfolio artifact table is started.
  • I committed my changes.

IST 03 Web Creator Project

  • code/index.html is included.
  • code/style.css is included.
  • My webpage has a title, heading, paragraph, and list.
  • My webpage has CSS styling.
  • A screenshot is saved in the images folder.
  • README.md is completed.
  • I committed my changes.

IST 04 EarSketch Music Project

  • code/earsketch-code.js is included.
  • A screenshot of my EarSketch project is saved in the images folder.
  • A playable audio file is saved in the audio folder.
  • The audio file is named clearly, such as song.mp3 or song.wav.
  • README.md is completed.
  • I committed my changes.

IST 05 micro:bit Hardware Project

  • code/microbit-code.js is included.
  • My project uses at least one input.
  • My project produces at least one output.
  • My project uses an event or conditional.
  • A screenshot or photo is saved in the images folder.
  • README.md is completed.
  • I committed my changes.

IST 06 Maqueen Robot Challenge

  • code/maqueen-code.js is included.
  • Team members are listed in the README.
  • Team roles are completed.
  • The robot uses movement.
  • The robot uses at least one input or sensor.
  • The code uses an event, loop, or conditional.
  • A screenshot or photo is saved in the images folder.
  • Testing and debugging reflection is completed.
  • The team committed final changes.

IST 07 Mini Game Project

  • code/index.html is included.
  • code/style.css is included.
  • code/game.js is included.
  • The game has a clear goal or objective.
  • The game uses player input.
  • The game has a score, win, lose, or feedback system.
  • A screenshot is saved in the images folder.
  • If used, audio is saved in the audio folder.
  • README.md is completed.
  • I committed my changes.

IST 08 Final Portfolio Showcase

  • README.md is completed.
  • My introduction is completed.
  • My best projects table is completed.
  • At least three project reflections are completed.
  • TSA connection is completed.
  • Career or pathway connection is completed.
  • Final reflection is completed.
  • I committed my changes.
